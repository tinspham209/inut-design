
## Blog Structure Overview

The blog content is organized in the `/blog` directory, where each blog post is stored as a markdown file (`.md` or `.mdx`). Each markdown file contains frontmatter metadata at the top, which includes essential information about the post such as:
- `slug`: a URL-friendly identifier for the post
- `title`: the title of the blog post
- `author`: the author's name
- `author_title`: the author's title or affiliation
- `author_url`: a URL to the author's profile or website
- `author_image_url`: a URL to the author's image
- `tags`: an array of tags relevant to the post
- `date`: the publication date of the post in ISO format

Here is an example of the frontmatter structure:

```markdown
---
slug: quy-trinh-dan-skin-laptop-chuyen-nghiep-da-nang
title: Quy Trình Dán Skin Laptop Chuyên Nghiệp Tại Đà Nẵng - 5 Bước Để Có Laptop Đẹp Hoàn Hảo
author: tinspham209
author_title: INUT Design
author_url: https://www.inutdesign.com
author_image_url: https://res.cloudinary.com/dmspucdtf/image/upload/v1663647671/inut/292635797_197003529328579_4330060878795101093_n_bjzhby.jpg
tags: [skin laptop da nang, quy trinh dan skin laptop, dan skin laptop chuyen nghiep, dich vu dan skin laptop da nang]
date: '2025-05-05T10:00:00.000Z'
---
```

After the frontmatter metadata, the description, truncated content, and table of contents are provided, following with this formatted content:

```markdown
Mô Tả Ngắn

<!-- truncate-->****

<!-- ## Table of contents -->
```

After that the content of the blog post is written in markdown format, allowing for rich text formatting, images, links, and other markdown features.

```markdown
Content of the blog post goes here...
```
